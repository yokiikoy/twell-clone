# クリーンルーム語表（Gemini / LLM レビュー用・背景）

この文書は、**手元の結合 TSV / JSONL**（`export_llm_review_bundle.py` の出力）と一緒にモデルへ渡すための**短い説明**です。コピーしてシステム／ユーザメッセージの冒頭に貼ってください。

## 語表がどうできているか

- **ビルド**: リポジトリの `packages/wordlists-v2` が **`pipeline`** サブコマンドで一括生成します。オープンコーパス（Tatoeba、JMdict、GeoNames、ja.wiktionary カテゴリ由来の koto 草案など）をマニフェストに従い取得し、**Sudachi 形態素付き JSONL** をマージ → レーンごとにゲートでフィルタ → 層化サンプル → エンジン側 `join-strokes` → **Web 向け JSON** として `twelljr-cr-*.json` を出力します（`--deploy` で `apps/web/public/cleanroom/` にコピー可能）。

- **各行の列**（結合ファイルの想定）:
  - **`lane`**: デッキ slug（例 `cr-jou1` … `cr-koto4`）。29 レーンの定義は `docs/cleanroom/lane-taxonomy-v1.md`。
  - **`surface`**: 表示用の表層形。
  - **`reading`**: ローマ字（typing 用・エンジンが付与）。
  - **`code`**: デッキ内 ID（例 `cr-jou1-0`）。
  - **`index`**: デッキ内の並び。

## 品質上の前提（レビュー時に重要）

- **辞書形・終止形の保証はしない**。特に **Jou（cr-jou1〜8）** は「**単一形態素**」かつ品詞 head がレーンに合うものだけを残すため、**連用形・語幹だけに見える表層**が混じり得ます（Tatoeba 文脈由来）。
- **人名・固有名・センシティブ語の除外は別工程**です。この語表はオープンデータの採取・ゲートの結果であり、**一般語としてふさわしいかの人間／LLM レビュー**を想定しています。
- **Kata**: カタカナ連続などのルール。JMdict の読み（reb）や地名の代替表記が混じると、見た目のノイズが出ることがあります。
- **Kata11**: 外国語地名レーン（`cr_kata_lane` 等）。データソース次第です。
- **Kan（cr-kan1〜6）**: ランタイムは `cr_kan_lane` 列が一致する行のみ。ガゼット未整備時はデッキが空になり得ます。
- **Koto**: Wiktionary のカテゴリメンバー（ページ題名）由来の行があり、**長さ・慣用性は一定ではありません**。

## データ規模と分割の目安

結合 1 ファイルは **数万行・数 MB** になり得ます。チャットへの全文ペーストより、**ファイルアップロード**や **lane 族ごとにファイルを分けて**複数回レビューする運用を推奨します。スクリプトは標準エラーに行数・バイト数を出します。

レビュー用の指示文テンプレートは同ディレクトリの [`GEMINI_REVIEW_PROMPT_ja.md`](GEMINI_REVIEW_PROMPT_ja.md) を参照してください。
