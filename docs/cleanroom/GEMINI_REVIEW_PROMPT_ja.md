# クリーンルーム語表・品質レビュー用プロンプト（Gemini 向け）

以下をユーザメッセージとして使い、**冒頭に `GEMINI_REVIEW_CONTEXT_ja.md` の本文を貼る**か、同内容を要約して渡してください。

---

## 入力データの扱い

{{TSV_OR_FILE_INSTRUCTION}}

- 各行は **1 エントリ**（語または短い句）。列 **`lane`**, **`surface`**, **`reading`**, **`code`**, **`index`** がある想定です。
- レーンの意味は `docs/cleanroom/lane-taxonomy-v1.md` に準じます（簡単に: `cr-jou*` は Sudachi 単一形態素＋品詞レーン、`cr-kata*` はカタカナ連続の行別、`cr-kan*` / `cr-koto*` はガゼット列によるマッチ、など）。

## あなたの仕事

次の **問題タイプ**に当てはまり**そうな**ものを、**優先度が高い順に最大 {{MAX_ITEMS}} 件**まで抽出してください。すべてを網羅する必要はありません。不確かな場合は「疑い」と明記してください。

1. **活用・語形**: 語幹や連用形などで終わっており、終止形・辞書形として提示するには不自然に見える。
2. **レーン矛盾**: その `lane` の定義（単一形態素、カタカナのみ、など）と**矛盾しうる**表層。
3. **固有名・センシティブ**: 人名・地名・団体名・不適切・センシティブに**見える**（断定せず疑いでよい）。
4. **記号・混在**: 記号・数字・異種スクリプトが**途中に**挟まっている、など。
5. **文脈依存**: 文から切り出されただけで、**単独では意味が取りにくい**。
6. **希少・難読**: 一般語彙として極端に稀、難読、**存在が疑わしい**表記。
7. **その他**: 上記以外の**違和感**。

## 出力形式（どちらか一方を指定）

- **Markdown 表**: 列 `lane`, `surface`, `reading`, `code`, `問題タイプ番号`, `理由（短文）`
- **JSON 配列**: 各要素 `{ "lane", "surface", "reading", "code", "category": 1-7 | "other", "reason": "..." }`

## プレースホルダの例

- `{{TSV_OR_FILE_INSTRUCTION}}` の例:
  - 「添付の `review.tsv` 全体を読み、上記の観点でレビューしてください。」
  - 「次の TSV を貼ります（1 行目はヘッダ）。…」
- `{{MAX_ITEMS}}` の例: `200`（モデルやトークンに合わせて調整）
